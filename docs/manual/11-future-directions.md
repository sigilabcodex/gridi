# 11. Future Directions

This chapter is about future, planned, and exploratory directions. It is not a list of finished features.

GRIDI's future work is expected to build on the same modular instrument model already present today: modules, subtypes/modes, routing, control/modulation, visual feedback, presets, sessions, and performance workflows.

## Areas already documented as future or refinement directions

### Stronger Control and Visual modules

Control and Visual modules already exist, but both families are expected to become richer. Planned/refinement directions include clearer modulation workflows, stronger control surfaces, additional visual modes, and more useful performance feedback.

### Expanded preset banks

Module and session presets exist now, but broader curated banks are a future direction. Documented ideas include more musical Drum, Synth, and GEN presets that help players learn and perform the instrument.

### Richer routing workflows

Routing exists now, including event, modulation, audio, and MIDI routing visibility. Future work may make routing clearer and more fluid while avoiding a giant DAW-style matrix unless the design changes intentionally.

### Interactive displays

Displays are intended to become behavior surfaces, not decoration. Future work may make generator and module displays more interactive and more directly tied to the sound or event behavior they represent.

### SONAR as a future pulse/echo mode

RADAR is implemented now. SONAR is reserved as a future GEN concept based on pulse propagation, distance, and echo-style semantics. It should not be treated as available in this version.

### Image-driven generation

A planned/exploratory GEN direction is image-driven generation, where an image is reduced into a compact internal field and scanned or interpreted musically. Documented traversal ideas include horizontal, vertical, diagonal, rotating-angle, and time-varying scans.

### Dataset/spreadsheet-driven generation

Another documented exploratory direction is dataset or spreadsheet-driven generation. The idea is to map lightweight tabular data into rhythm, pitch, probability, or event structures. This direction is described as data-driven generation, not as an AI/ML analytics workflow.

### Quantum/Schrödinger-inspired generation

GRIDI documentation also mentions conceptual quantum or Schrödinger-inspired generation. This is an artistic metaphor direction around ambiguity, branching, observation, and collapse-like musical behavior. It is not a claim of scientific quantum simulation.

### MIDI expansion

Current GRIDI includes live Web MIDI input foundations and MIDI routing visibility for Synth targets. Future MIDI directions may include output, richer mappings, channel policies, CC/aftertouch/MPE-style expansion, and deeper device routing.

### Live coding exploration

Live coding is documented as a longer-term possibility. If explored, it should support GRIDI's instrument identity rather than turn the app into a separate coding environment first.

### Sampling, looping, and granular ideas

Sampling, looping, and granular or grain-based directions are documented as longer-term ideas. They are exploratory and not part of the current baseline workflow.

### Mobile and global controls refinement

GRIDI's compact header/global controls are still being refined, especially for narrow and mobile layouts. Future work may use a more dedicated mobile/global controls pattern so transport, session, routing, and utility actions do not compete with the module workspace.

## The important promise

Future GRIDI should still feel like GRIDI: a modular generative instrument you can assemble, route, listen to, and perform. New ideas should make that instrument deeper without pretending that planned concepts are already finished.

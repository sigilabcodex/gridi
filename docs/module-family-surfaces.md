# Module family surface redesign

This pass intentionally changes **visible module anatomy** so family identity reads before labels.

## Structural differences by family

| Family | Primary interaction surface | Layout structure | Information rhythm | Visible emphasis |
|---|---|---|---|---|
| Trigger | Pulse map + step activity rail | Pulse rail on top, generative controls below, raw preview secondary | Fast scan from pulse cells to density/subdiv controls | Rhythm, gate density, transport relation |
| Drum | Percussive shaping lanes | 3 lane split: **Transient**, **Body + pitch**, **Performance** | Left-to-right shaping path from attack/noise into body then output | Transient punch and compact performance control |
| Tonal/Synth | Tone architecture strips | 3 strips: **Oscillator + motion**, **Envelope**, **Filter + tuning** | Timbre first, articulation second, final tone/output third | Oscillator/filter/envelope modulation workflow |
| Visual | Reactive display-first panel | Large display frame, small control dock underneath | Read activity first, tweak mode/settings second | Visual activity and monitoring continuity |
| Utility/controller | Routing card in secondary panel | Dedicated routing card with map/readout in Routing tab | Connection mapping as relationship data, not synthesis params | System relationships and trigger-to-voice mapping |

## Notes

- Tabs remain secondary; each family uses a different **main-panel anatomy**.
- Distinction is carried by structure and grouping, not just color/icon/labels.
- Workspace footprint and lane model are unchanged.
